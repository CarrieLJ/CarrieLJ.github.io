HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • The section semantic element defines a section in a document that contains thematic content.
  • The tag refers only to what appears inside the angle brackets.
  • The p means paragraph.
  • The ul means unordered list; ol means ordered list; li means list items.
  • The src defines location of an image.
  • The br creates a line of empty space.
  • The a (which means anchor) creates a link to the same webpage or other webpages.
  • The href is an attribute that points to the URL for the link.
  • The alt is an attribute that contains a text string that describes two apperances and functionality of the image.
  • The rel is the relationship between current document and the linked document.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS is styling an element directly in the HTML file.
  • Internal Style sheet is embeded on an entire stylesheet directly in the HTML file using a style element in the head element (this will make ALL h1 elements the same)
  • External CSS stylesheet allows developers to keep all their CSS rules in a separate file (this is preferred and makes design easier).
  • The selector defines the element or attributes to which the rules or declarations will apply.
  • The declarations contain 2 components: propery (ex: color) and the value of the property (ex:blue).
  • * is the wild card symbol which adds to all elements on the page.
  • A comma between elements will add same styling to all those elements.
  • DRY = Do Not Repeat Yourself
  • The block assigns each image to start on a new line.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • Git Flow
      1. Make sure you are on the branc within command line by: git status Add the changed files in the working directory to the stagging area of Git by entering: git add -A
      2. Changes have been staged, now commit to our repository by entering: git commit -m "added CSS to project" (for example)
      3. Verify branch is up to date before opening a pull request: git pull origin main
      4. Push to feature branch entering: git push origin feature/add-css
      5. Open a pull request within GitHub by clicking Pull Requests on the prework-study-guide repository. Then click "New pull request"
      6. Verify base branch is main and to comparison branch is the feature/add-css branch (for example). Add a title with a short description of the changes. Click "Create pull request" Merge your changes with the main branch by selecting "Merge pull request"
      7. Close another GitHub issue
      8. Update your local main branch by entering: git checkout main
      9. Then pull the remote main branch into the local main branch by entering: git pull origin main
